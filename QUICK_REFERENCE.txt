SETUP
─────
pip install -r requirements.txt
export OPENAI_API_KEY="sk-..."
brew install tesseract  (or: apt-get install tesseract-ocr)

QUICK START
───────────
from pipeline import DocumentPipeline, PipelineConfig

config = PipelineConfig.from_yaml("config/pipeline_config.yaml")
pipeline = DocumentPipeline(config)
result = pipeline.process("invoice.pdf")

print(result.extracted_fields)       # Dict with confidence scores
print(result.requires_review)        # Route to human review?
print(result.pii_detected)           # PII found + redacted?
print(result.confidence_summary)     # Min/avg/max confidence

BATCH PROCESSING
────────────────
results = pipeline.process_batch(["doc1.pdf", "doc2.pdf", "doc3.pdf"])

CONFIGURATION
──────────────
Edit config/pipeline_config.yaml:
  quality_gate.min_quality_score: 0.75
  llm.model: "gpt-4o-mini"
  confidence.field_review_threshold: 0.80
  pii.entities_to_detect: [PERSON, EMAIL, PHONE, IBAN, TAX_ID]

PIPELINE STAGES
───────────────
1. Quality Gate          → Score image quality (0-1)
2. OCR                   → Extract text
3. Language Detection    → Identify language
4. PII Detection         → Find sensitive data
5. PII Redaction         → Replace with [REDACTED]
6. LLM Extraction        → Structured field extraction
7. Confidence Scoring    → Per-field confidence (0-1)
8. Validation & Routing  → Flag for human review if needed

CUSTOMIZATION
──────────────
Add PII pattern:
  pipeline/pii/detector.py → PATTERNS dict

Add document type:
  1. Add schema in validation/schema.py
  2. Add prompt in extraction/llm_extractor.py
  3. Update config with new prompt_version

PROJECT STRUCTURE
──────────────────
pipeline/
├── quality_gate/       Quality classifier
├── ocr/               Text extraction & language detection
├── pii/               PII detection & redaction
├── extraction/        LLM extraction & confidence scoring
├── validation/        Schema validation & routing
├── config.py          Configuration dataclasses
└── pipeline.py        Main orchestrator

tests/
├── test_pii_redaction.py
├── test_confidence.py
└── test_validation.py

KEY FEATURES
────────────
✓ Quality gating before expensive LLM processing
✓ Per-field confidence scoring for focused review
✓ PII detection & redaction (audit logged, never exposed)
✓ Prompt versioning for reproducibility
✓ Human-in-the-loop routing (routes to review when uncertain)
✓ Language detection (multilingual support)
✓ Schema validation (Pydantic)
✓ Batch processing
✓ Comprehensive tests & documentation

LIMITS & FUTURE
────────────────
• Quality classifier uses heuristics (train CNN in production)
• Invoice extraction only (extensible to other doc types)
• No persistence layer (add database as needed)
• No job queue (add Celery for scale)
• No Prometheus monitoring (integrate as needed)
